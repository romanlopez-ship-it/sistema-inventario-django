# Product Backlog - Sistema de Inventario
**Roles:** Product Owner (Docente), Scrum Master, Equipo de Desarrollo.

| ID | Historia de Usuario | Prioridad | Puntos (Fibonacci) |
|:--- |:--- |:--- |:--- |
| **HU-01** | Como encargado de bodega, quiero **ver una lista de productos** para conocer el stock disponible actualmente. | Alta | 2 |
| **HU-01a** | Como administrador, quiero **ver el detalle de un producto** específico para consultar sus características completas. | Alta | 2 |
| **HU-02** | Como encargado de compras, quiero **registrar un nuevo producto** para mantener actualizado el catálogo de la tienda. | Media | 3 |
| **HU-03** | Como administrador, quiero **editar los datos de un producto** para corregir errores de información o cambios de precio. | Media | 3 |
| **HU-04** | Como jefe de inventario, quiero **eliminar productos obsoletos** para mantener la base de datos limpia de artículos descontinuados. | Baja | 5 |
| **HU-05** | Como administrador del sistema, quiero un **sistema de autenticación (login/logout)** para proteger la información sensible del inventario. | Alta | 8 |

## Definición de Roles del Equipo
*   **Product Owner (PO):** Docente (Maximiza el valor del producto) [6].
*   **Scrum Master (SM):** Estudiante (Garantiza el proceso y elimina impedimentos) [6].
*   **Equipo de Desarrollo:** Estudiante (Autoorganizado para entregar incrementos de software) [6].

## Notas de Estimación
Se utiliza la **Escala Fibonacci** (1, 2, 3, 5, 8, 13) para medir la complejidad relativa. Los valores más altos indican mayor incertidumbre o esfuerzo técnico requerido [4, 7].
